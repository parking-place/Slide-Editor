# v0.10.0
Date: 2026-04-09

## Added

- 구버전 JSON 임포트와 구버전 프로젝트 로드 시 이미지를 먼저 표시한 뒤, 순차적으로 WebP로 backfill 변환하고 완료 알림을 띄우는 호환 로직을 추가했습니다.
- 프로젝트별 비동기 WebP 이미지 업로드 API와 이미지 자산 상태 조회 API를 도입했습니다.
- 이미지 업로드 직후 슬라이드 미리보기에서 변환 중 상태를 표시하고, 완료되면 자동으로 WebP 이미지를 반영하는 UI를 추가했습니다.
- 시스템 `node` 또는 저장소의 포터블 Node로 `scripts/server.js`를 바로 실행할 수 있는 로컬 서버 시작 배치 파일을 추가했습니다.

## Changed

- 에디터 미리보기에서 별도 목차 슬라이드를 제거하고, 좌측 Navigator만 남겨 가이드와 같은 탐색 구조로 정리했습니다.
- 더 이상 사용하지 않는 PPTX 다운로드 버튼, 변환 로직, 테마 내 PPTX 전용 설정을 제거해 HTML 가이드 중심 구조로 단순화했습니다.
- 에디터, 가이드, 프로젝트 매니저, 테마 모달의 스크롤 영역에 테마 색상을 따르는 얇은 커스텀 스크롤바를 적용했습니다.
- 브랜딩, JSON 임포트, 백업 기능을 프로젝트 매니저로 통합하고 헤더의 중복 관리 경로를 정리했습니다.
- 브랜딩의 프로젝트명은 항상 현재 프로젝트 저장 이름과 동기화되도록 정리했습니다.
- JSON 임포트 시 파일 내 브랜딩 프로젝트명을 우선 사용해 서버 프로젝트로 저장하도록 변경했습니다.
- JSON 백업 파일명은 `프로젝트명_data_YYMMDDhhmmss.json` 형식으로 저장되도록 정리했습니다.
- 저장 데이터와 백업 JSON에 `savedVersion`을 기록하고, 프로젝트 매니저에서 저장 버전을 표시하도록 개선했습니다.
- 버전 정보가 없는 구버전 데이터는 프로젝트 매니저에서 `old`로 표시하도록 정리했습니다.
- `src` 구조를 기능 모듈 중심으로 재정리하고, CSS도 `src/styles` 하위 파일로 분리했습니다.
- HTML5 `dialog`, `form`, `fieldset`, `details`, `progress`, `output`, `template` 기반 UI 보강을 반영했습니다.
- HTML 가이드 출력 구조를 HTML5 시맨틱 구조와 outline 메타데이터 기준으로 정리했습니다.
- 가이드 HTML 다운로드 기본 파일명을 현재 프로젝트명 기준 `.html` 형식으로 바꾸었습니다.
- `README.md`와 `docs/slide_editor_architecture.md`를 현재 HTML 가이드 중심 구조에 맞게 최신화했습니다.

## Fixed

- 이미지 업로드 직후 변환 완료 전까지 이미지 영역이 비어 보이던 문제를 상태 박스로 보완했습니다.
- 가이드 보기에서 네비게이터 현재 위치 표시와 이미지 확대 기능 회귀를 복구했습니다.
- 가이드 네비게이터를 에디터와 같은 구조로 맞춰 크기와 탐색 동작을 안정화했습니다.
- 프로젝트 매니저의 `Selected Project` 상세 텍스트가 테마 색을 따르지 않던 문제를 수정했습니다.
- 에디터 헤더의 프로젝트명 표시에서 내부 디렉터리명(`project id`)이 함께 노출되던 문제를 정리했습니다.
- 같은 이름의 프로젝트를 JSON으로 임포트할 때 자동으로 `_2`, `_3` suffix를 붙여 저장하도록 수정했습니다.
- HTML 다운로드 시 깨진 이미지 경로가 있어도 fallback을 시도하고, 실패한 이미지만 제외한 채 다운로드를 계속하도록 보강했습니다.
